# AI PR Review 助手 — 核心要求

请开发一个AI代码评审工具，帮助开发者提升Pull Request的Review效率与质量。

## 要求

请了解开发者在代码评审中的真实需求，设计并实现一个以 AI 辅助分析为核心的工具。用户指定 GitHub PR，系统自动获取代码变更并智能分析，辅助发现问题。

需支持：

- **PR 变更总结** — 对 PR 中的代码变更进行归纳概括，帮助 reviewer 快速理解改动范围和意图。
- **风险代码识别** — 自动检测潜在的安全漏洞、性能问题、逻辑错误、代码规范违规等风险点。
- **Review 建议生成** — 基于分析结果生成具体的、可操作的 review 建议，辅助 reviewer 决策。

## 关键评估因素

请综合考虑以下方面：

- **分析准确性** — 减少误报（false positive）与漏报（false negative）。
- **上下文理解** — 不仅分析 diff 本身，还需理解代码变更的上下文（文件结构、依赖关系、项目背景等）。
- **误报与漏报控制** — 平衡检出率与精准度，提升 reviewer 对 AI 建议的信任度。
- **响应速度** — 在保证分析质量的前提下，控制响应延迟。
- **使用体验** — 界面友好、报告清晰、易于集成到现有工作流中。

## 设计说明要求

请在作品中说明系统在以下方面的设计思路：

- **模型选择** — 为何选择某种/某几种模型，如何平衡效果与成本。
- **上下文获取方式** — 如何获取和理解 PR 的完整上下文（文件级、仓库级等），避免脱离语境的分析。
- **未来扩展方向** — 系统可能的演进方向，如支持更多代码托管平台、增量学习、自定义规则引擎等。
